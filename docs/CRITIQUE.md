## 業主監造

> 產品畫面看得出是真的有在跑的系統，斷網續傳、日報鎖版、告警附照片這幾點有打到工地實務，如果我是承商工安主管大概會約。但我是業主端監造，管三個公共標案，我的問題是「能不能把這東西寫進招標文件、要求承商導入、然後拿它的紀錄當驗收與查核佐證」——整個網站從頭到尾都在跟工地主任講話，沒有一句話是講給我聽的：沒有業主唯讀帳號、沒有證據力機制說明、沒有資料歸屬條款、沒有 API 與報表格式，連「已鎖版」三個字都沒解釋鎖了什麼。更麻煩的是你們自己宣稱「本頁畫面皆擷取自真實系統」，結果 CCTV 宮格時間戳是 13:02 白天、畫面卻是紅外線夜視，hero 又顯示 01:53——我第一眼就在懷疑你們的時間戳，這對一個要賣「紀錄可信」的平台是自打臉。所以我不會自己約這 15 分鐘 demo，最多把連結轉給承商參考；要讓我親自出席，官網得先回答下面這些。

- **[deal-breaker] 紀錄證據力零說明：日報標「已鎖版」、告警標「附照片存證」，但沒有任何機制描述（時間戳來源、鎖版後可否改、誰改過留不留軌跡、有無雜湊/簽章），而且官網截圖本身時間戳互相矛盾——CCTV 宮格顯示 13:02（白天）卻是紅外線夜視畫面、hero 戰情室顯示 01:53:36，同頁不同元件對不上**
  - 為什麼卡住：我要拿這些紀錄當驗收佐證、勞檢與工程會查核回應、甚至日後保固爭議的證據。時間戳能不能被承商事後竄改，直接決定這平台對業主有沒有價值；官網自己的畫面都對不齊時間，等於當場示範了「這時間戳不可信」
  - 要怎麼說服：加一個「紀錄怎麼成為證據」區塊：鎖版機制（鎖版後唯讀、異動留審計軌跡）、時間源（NTP/伺服器時間）、照片與告警的完整性保護方式；官網截圖時間統一，或老實標註「畫面為凌晨實拍、部分數字為示意」——你們敢寫凌晨拍的夜視是真畫面，就該把時間也對齊
- **[deal-breaker] 「業主要求承商導入」這條路徑完全不存在：全站敘事對象是承商（工安主管手機收通知、工地主任開早會），沒有業主/監造唯讀角色、沒有多方權限說明、沒有跨標案視角；連預約表單都只有「姓名/公司/電話/工地規模」，沒有身分欄位讓我表明我是監造不是承商**
  - 為什麼卡住：公共工程是三方結構（業主-監造-承商）。我要求承商導入的前提是我自己看得到資料且承商動不了我看到的東西；集團方案講的是營造廠總部看自己的案場，跟業主監督是兩回事。現在這網站等於叫我去幫承商買一套我自己進不去的系統
  - 要怎麼說服：加「給業主與監造」的區塊或獨立頁：角色權限矩陣（業主唯讀、監造可註記、承商操作）、一個業主帳號看多個承商標案的畫面截圖、表單加「我是業主/監造/承商/整合商」選項——有這個我才有理由親自出席 demo
- **[major] 資料歸屬與保存政策空白：合作卡寫「按工地訂閱，工期結束即止」「撤場設備可移轉下個案場」，設備的去向講了，資料的去向隻字未提——完工後影像、告警、日報歸誰？保存幾年？可否整包匯出？人臉與車牌個資怎麼刪？**
  - 為什麼卡住：公共工程保固期動輒數年，職災訴訟常在完工後才開打；「訂閱結束即止」在我耳裡就是「完工那天證據一起消失」。加上人臉/車牌是特種個資，業主要求承商導入等於業主一起背個資風險，FAQ 那題「影像和人員資料放在哪裡」還是收合的看不到答案
  - 要怎麼說服：白紙黑字寫資料歸屬摘要：完工可完整匯出（列格式）、保存年限與加購選項、合約終止後的刪除流程與個資處理聲明；FAQ 個資與資料存放兩題的答案直接攤開，不要收合
- **[major] 與公共工程制度的對接幾乎沒講：只有一行小字「偵測項目照職安署寫法」和一題收合的「這些紀錄能用在勞檢或工安查核嗎？」；沒提三級品管、工程會查核、金質獎佐證用途，日報只說「主管機關備查」但沒說對應哪張法定表單**
  - 為什麼卡住：我要把平台寫進招標文件，需要的是「本平台輸出可對應○○表報/○○查核項目」這種可引用的規格文字；一句「照職安署寫法」太薄，查核委員不會買單
  - 要怎麼說服：做一頁「查核與備查對接」：日報欄位對應的法規表單、哪些輸出可作查核佐證、最好附一段可直接抄進招標規範的功能規格文字（spec-in）——這頁做出來，我就是你們的免費業務
- **[major] 資料出口沒講：系統串接圖六個系統全是「進來」（門禁、環測、CCTV、廣播），沒有任何「出去」的箭頭——沒有開放 API、webhook、CSV/JSON 匯出說明，報表只有 PDF 一種格式**
  - 為什麼卡住：業主端有自己的工程管理平台，我要的是承商資料能自動匯進我的系統做跨標案比較，而不是每天收一份 PDF 用眼睛看；集團方案講「與既有 ERP/門禁制度整合」證明技術上做得到，但那是賣給營造廠的，業主端的資料介接完全沒提
  - 要怎麼說服：公開 API/匯出能力清單（哪些資料可用 API 撈、支援格式、webhook 事件），不用完整文件，至少一張「資料出得去」的示意——你們架構圖的 hub 現在只進不出
- **[minor] 信任帶沒有可查證的主體與實績：頁尾只有四句口號（台灣開發·資料不出境、免換設備、斷網續傳、對接職安卡），沒有公司名稱、統編、實績案場數，也沒有客戶案例**
  - 為什麼卡住：我在公家體系做事，要求承商採購一個官網上連公司登記名稱都找不到的平台，簽辦根本送不出去；沒有案例我可以理解（新產品），但連「你是誰」都沒有就是另一回事
  - 要怎麼說服：頁尾放公司全名、統編、聯絡地址；案例還沒有的話，用可查證的中性述句代替（例如「目前於 N 個工地運行中」），不用造假，但要給我一個能寫進簽呈的主體
- **[minor] 示範數據自己打自己：AI demo 截圖裡「未處理告警 204 件」「平均回應時間 18 時 41 分」——這是一個告警堆積如山、快 19 小時沒人理的工地畫面**
  - 為什麼卡住：你們前面才說「3 秒內從偵測到通知」，後面就展示告警平均擺 18 小時；對監造來說這正是我最怕的場景：平台裝了、告警狂跳、承商麻痺不處理，出事後這些「未處理」紀錄反而變成業主連帶督導不周的證據
  - 要怎麼說服：demo 畫面換成健康營運狀態（未處理個位數、回應時間分鐘級），或在旁邊講清楚告警分級與結案 SLA 機制——告訴我這平台怎麼防止告警疲乏，而不是展示它

## 競品業務

> 說實話，這個站做得漂亮，敘事也準——「工地那麼大，人只有一雙眼睛」這種話比我們原廠型錄會講人話。但客戶把它丟給我問差在哪，我十分鐘就能把這場 demo 拆掉：整個網站從頭到尾找不到公司名稱、統編、地址、一個客戶、一組可驗證的數字。我只要當著客戶的面問三句——「3 秒是誰量的？」「AI 是誰訓練的、誤報率多少？」「系統掛了誰來修、幾小時到場？」——網站上一個答案都沒有，然後我再把他們自己的 hero 截圖放大：「未處理告警 204 件、在場人數 0 人」，這就是他們所謂真實運行中的系統，告警堆到 204 件沒人處理。最後補一刀：你要把全工地人臉和影像交給一家連官網都查不到是誰的公司？所以結論是：客戶約他們 demo 我根本不擋，因為現階段這個站替我打工——它把期待拉滿、把證據留白，demo 完客戶自然回頭找有原廠保固和 SLA 的我們。但我也得跟公司回報：這家人對台灣工地的痛點抓得很準（職安卡、勞檢備查、斷網續傳、免換設備），等他們補上客戶案例、SLA 和公司身分，就不好打了，最好趁現在簽走大客戶。

- **[deal-breaker] 公司身分零揭露：全站沒有公司名稱、統編、地址、電話、Email、團隊背景、成立年份，頁尾等於空白。**
  - 為什麼卡住：我在客戶面前只要說一句「你 Google 得到這家公司是誰嗎？」就結束了。工地要把人臉、影像、進出紀錄交給一個查不到登記資料的主體，採購和法務第一關就打回票；對照我們原廠＋在地代理雙實體，信任差距是結構性的。
  - 要怎麼說服：頁尾放公司全名／統編／地址／聯絡電話與 Email，加一頁「關於我們」：團隊工地／系統整合資歷、成立時間、辦公室照片。不用吹牛，只要「查得到、找得到人」就守住了。
- **[deal-breaker] 零客戶證據：沒有任何客戶 logo、案例、導入數量、使用時長、成果數字，卻反覆強調「畫面皆擷取自真實運行系統」。**
  - 為什麼卡住：「真實系統」跑在哪個工地、跑了多久、幾個案場在用？講不出來我就替客戶下結論：「他們的真實系統就是自己辦公室那一套」。B2B 採購沒有 reference site 幾乎無法立案。
  - 要怎麼說服：不能放 logo 就放匿名案例卡：「桃園某社宅新建工程・N 路攝影機・上線 M 個月・告警 X 件全數結案」，配一句可具名或匿名的工地主任引言；哪怕只有一個 pilot 也要寫出來，並開放 demo 時看 live 系統。
- **[deal-breaker] SLA／保固／維運承諾全站缺席：沒有 uptime 目標、故障到場時效、備品更換政策、保固年限、維運人力說明。導入三步只寫「全程支援」四個字。**
  - 為什麼卡住：工地是 24 個月的工期，攝影機淋雨、閘道被挖斷是日常。我方標配「4 小時到場、次工作日換機、N 年保固、原廠備品庫」，我只要問「你們閘道壞了誰來修？」網站無法回答，工務經理就會自己想像最壞情況。
  - 要怎麼說服：加一個「維運與承諾」區塊或 FAQ：平台 uptime 目標、告警通道冗餘（LINE＋簡訊備援）、設備故障回應與到場時效、備品政策、保固範圍。數字可以保守，但不能沒有。
- **[major] 「3 秒內從偵測到通知」無量測定義：不知道誰量的、幾路併發、什麼網路條件、是平均還是最差值、含不含 LINE 推播延遲；且斷網時此承諾自動作廢。**
  - 為什麼卡住：這是全站最顯眼的數字，也是最好打的一根：我就問「合約敢不敢寫 3 秒、超過怎麼罰？」量測方法拿不出來，客戶會把整頁其他數字一起打折。
  - 要怎麼說服：加註腳寫清楚：「端到端實測（AI 判定→App／LINE 收到），30 路併發、4G 上行、P95 < 3 秒，量測方法見 demo」，並說明斷網時走現場廣播／警示燈的本地告警，恢復後補送。敢定義就守得住。
- **[major] AI 偵測是黑箱：不講用什麼模型、誰訓練、用多少台灣工地資料、誤報率／漏報率多少；頁面上的「模擬一次偵測」是腳本動畫，不是真推論。**

  - 要怎麼說服：不必公開模型細節，但要給可驗證的承諾：實測 precision／recall 區間、誤報如何一鍵回饋與再訓練、以及殺手鐧——「兩週免費 PoC，用你自己工地的攝影機驗證誤報率」。把賭注推回現場，我就沒話講。

  - 為什麼卡住：這張圖我會直接投影給客戶看：204 件沒人處理，代表要嘛誤報氾濫到處理不完，要嘛根本沒人在用這套系統——兩個結論都致命，而且素材是他們自己提供的。

  - 要怎麼說服：列「已驗證整合」清單：協定（RTSP／ONVIF profile、Wiegand／OSDP、Modbus）＋已實測品牌型號，區分「已驗證」與「支援協定理論可接」。誠實的短清單比模糊的全能宣稱有力。
