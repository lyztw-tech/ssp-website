# DECISIONS.md — 決策紀錄

> 記「為什麼這樣做」，避免之後重複討論已經拍板的事。一條一個決策，**最新的放最上面**。

## 2026-07-12 — Header 改用「同心圓＋雷達 ping」的自製 logo 造型
- **決定**：換掉純 unicode `◉` 塞色塊的舊 logo，改做 `NavMark`（同心圓光圈＋teal→blue 漸層核心＋會 blink 的訊號點＋雷達式 ping）。
- **考慮過的選項**：維持簡單色塊 logo／做一個呼應「即時監測」語意的自製圖形。
- **理由**：使用者反饋原本 nav「有點醜」、要求「有點特色感」；新造型呼應全站「即時監測／LIVE」的視覺語言，不是憑空加裝飾。
- **取捨**：手機窄螢幕（≤760px）版面更擠，需要額外處理排版（見下一則決策）。

## 2026-07-12 — 手機版隱藏深淺色切換鈕
- **決定**：≤760px 窄螢幕直接隱藏深淺色切換鈕（`.theme-btn{display:none}`）。
- **考慮過的選項**：縮小切換鈕塞進同一行／直接隱藏。
- **理由**：字級調大＋新 logo 造型後，窄螢幕會擠成三行爛版（logo 換行、切換鈕換行、CTA 換行）；深淺色切換鈕本來就是給桌機做比較評估用的次要功能，非核心。
- **取捨**：手機用戶無法自行切換深淺色主題（僅桌機可切換）。

## 2026-07-12 — 深淺色主題：產品展示框與夜間值班區鎖定深色 token
- **決定**：加入深淺色主題切換（`localStorage` 記憶），但「產品展示框」(`.mod-frame`) 與「夜間值班區」(`.night`) 在淺色模式下仍鎖回深色 token。
- **考慮過的選項**：全站含產品展示框都跟著切換淺色／產品展示框固定深色。
- **理由**：這兩處是在展示「產品畫面」本身（深色 UI 是產品的實際外觀），跟著切換淺色會讓展示畫面失真；效果是「淺色頁面、深色產品畫面」。
- **取捨**：鎖色區塊的 CSS 要額外處理（`color` 在 body 層就解析完，鎖 token 的區塊要重新宣告 `color: var(--text)`），增加維護複雜度。

## 2026-07-12 — 全站字級分兩輪調大
- **決定**：body 基準字級 17→18px，nav/表單/方案/ROI 等關鍵文案再加大，但刻意排除「六大模組活元件」與「門禁雙劇場」demo 的縮小版真實產品 UI。
- **考慮過的選項**：一次調到底／分輪調整並觀察回饋。
- **理由**：使用者反饋「字不要太小，主管都有年紀了」；第一輪加大 1–2px 後使用者仍覺得要再大，故第二輪再加大一輪。縮小版真實產品 UI 字本來就該小（跟截圖同理），不在調整範圍。
- **取捨**：`globals.css` 逐行手動核對後才替換（非正規式亂改），共動了約 140 處 `font-size`，改動量大、需要仔細核對避免誤改到 demo 元件的字級。
